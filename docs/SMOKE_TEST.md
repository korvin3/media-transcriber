# Smoke Test Protocol (Phase 8)

## Цель
Подтвердить, что релизный билд стабильно транскрибирует длинный файл (3 часа) на macOS и Windows.

## Тестовые входные данные
- 1 медиафайл длительностью ~3 часа (`mp4` или `mkv`).
- Валидная модель `whisper.cpp` (`.bin` или `.gguf`).

## Сценарий
1. Установить свежий релизный артефакт из `build/bin/`.
2. Запустить приложение.
3. Проверить diagnostics:
   - `ffmpeg`, `ffprobe`, `whisper.cpp` = `pass`
   - model path и output dir = `pass`
4. Выбрать 3-часовой файл и запустить транскрибацию.
5. Во время выполнения проверить:
   - UI не зависает
   - события стадий приходят (`preprocessing`, `transcribing`, `exporting`)
   - доступны логи команд
6. Дождаться результата и проверить:
   - статус `done`
   - создан `.txt` в output directory
   - содержимое файла непустое
7. Повторить с отменой:
   - запустить вторую сессию
   - нажать Cancel
   - получить статус `cancelled`

## Критерии PASS
- End-to-end транскрипция 3-часового файла завершается без краша.
- Текстовый файл экспортируется корректно.
- Отмена работает и не блокирует последующие запуски.

## Логирование результата
- Создать файл отчёта по шаблону: `docs/smoke-results/TEMPLATE.md`.
- Сохранить отчёт как `docs/smoke-results/YYYY-MM-DD-<os>-smoke.md`.
